# Aru Adoption Plan

This plan turns Aru from a private language project into a public learning and writing project.

The stable language core remains **Aru v1.0.0**. Adoption work should increase use without changing the core.

## Adoption Loop

```text
learn fast -> write something real -> get checked -> publish examples -> invite the next learner
```

## Audience

Aru should first serve people who want:

- a small constructed language they can learn quickly;
- examples that are easier than natural-language grammar;
- a clear writing system for short thoughts;
- a public project where contributions are structured.

## Public Entry Points

Use these entry points in order:

1. `README.md` for the project overview.
2. `index.html` for the public portal and playground.
3. `COURSE.md` for structured learning.
4. `COMMUNITY_CHALLENGES.tsv` for first public tasks.
5. `TEXT_SUBMISSIONS.tsv` for draft contributions.
6. `TRANSLATION_BENCH.tsv` and `QUALITY_METRICS.md` for review.

## Growth Targets

The project should maintain:

- a stable core;
- at least 150 phrasebook entries;
- at least 100 corpus texts;
- at least 30 dialogues;
- at least 50 translation benchmark rows;
- at least 30 public community challenges;
- a release gate that validates every public artifact.

## What Not To Do

- Do not change grammar to make a single challenge easier.
- Do not add roots before trying compounds or short rephrasing.
- Do not optimize for poetic style before ordinary clarity.
- Do not make contribution steps depend on private knowledge.

## Success Signal

A useful adoption milestone is reached when a new learner can:

```text
read 10 examples -> write 5 sentences -> submit 1 draft -> understand review notes
```
