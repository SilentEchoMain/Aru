# Code of Conduct

Aru is a public language project. Contributions should make the project easier to learn, use, and improve.

## Expected Behavior

- Be clear and constructive.
- Assume good faith while asking for evidence.
- Critique proposals, not people.
- Keep discussions focused on examples, compatibility, and user needs.
- Help new contributors understand Aru v1.0.0 before proposing large changes.

## Unacceptable Behavior

- Harassment, insults, threats, or personal attacks.
- Repeated disruption of issue or review threads.
- Dismissing compatibility concerns without examples.
- Posting unrelated promotional or abusive content.

## Project Norms

Language design can be subjective. Aru handles disagreement through examples:

```text
problem -> attempted expression -> proposal -> compatibility note
```

When a discussion gets stuck, reduce the claim to a short Aru text and test whether existing grammar can express it.

## Enforcement

Maintainers may close, lock, edit, or remove contributions that make the project unsafe or unusable.

The goal is not punishment. The goal is to keep Aru useful, readable, and welcoming.
